Research Design

Six Hypothesis Tests

We apply comprehensive phonetic analysis to 50,000+ university professors across three institutional tiers:

H1: Name Sophistication ‚Üí University Prestige

Prediction: Higher syllable count, phonetic complexity, and uniqueness correlate with higher-ranked universities
Method: Ridge regression predicting US News ranking
Shocking if: R¬≤ > 0.10 (names explain 10%+ of prestige variance)
Real-world meaning: "Sophisticated" names like 'Worthington' or 'Montgomery' cluster at top universities as much as SAT scores cluster‚Äîsuggesting phonetic patterns in academic hiring we don't consciously acknowledge.

H2: Phonetic Authority ‚Üí Academic Rank

Prediction: "Authoritative" sounding names predict full professor/distinguished status
Method: Logistic regression with authority score, consonant hardness
Shocking if: Odds Ratio > 1.5 with ROC AUC > 0.75
Real-world meaning: Names with "gravitas" (like 'Bradford' or 'Thornton') achieve tenure 50% more often than softer-sounding equivalents at same publication rate‚Äîthe phonetic version of "looking the part."

H3: Memorability ‚Üí Research Impact (h-index)

Prediction: Easy-to-remember names accumulate more citations (name recognition effect)
Method: Ridge regression with Google Scholar metrics
Shocking if: R¬≤ > 0.15 (citation advantage for memorable names)
Real-world meaning: 'Chen' gets cited more than 'Sczcepanski' at identical paper quality‚Äîthe academic equivalent of brand name recognition. Easier to spell = easier to cite = higher h-index, revealing bias in how credit flows.

H4: Field-Specific Name Patterns

Prediction: STEM professors have "harder" phonetics (plosives) vs Humanities (softer vowels)
Method: Independent t-tests + ANOVA across fields
Shocking if: Cohen's d > 0.5 (large effect size across multiple features)
Real-world meaning: Physics departments sound like 'Krantz, Brock, Steiger' while English departments sound like 'Emilia, Lily, Sophia'‚Äîdifference as pronounced as height gaps between genders, suggesting fields self-select by sound.

H5: Top-20 University Prediction (THE BIG ONE)

Prediction: Name phonetics alone predict Harvard/MIT/Stanford vs other institutions
Method: Logistic regression with full 10+ feature set
Target: ROC AUC > 0.85 (match hurricane 0.916)
Paradigm-shifting if: AUC > 0.90 (diagnostic-grade prediction of elite universities)
Real-world meaning: If we hit AUC > 0.90, a blind algorithm hearing only names‚Äîno CVs, no publications‚Äîwould identify Harvard professors as accurately as a strep throat test identifies infection. That would mean academia's "meritocracy" has a phonetic signature as detectable as medical conditions, revealing systematic patterns in who reaches elite positions. Not "names cause Harvard jobs," but "Harvard selects certain name patterns" with diagnostic precision‚Äîlike how certain dog breeds cluster in certain neighborhoods.

H6: Gender-Name Interaction (Exploratory)

Prediction: Masculine/neutral names provide rank advantage (exploratory, sensitive)
Method: Interaction term analysis
Status: Deferred to follow-up study with proper gender data

How to Run the Analysis

Step-by-Step Execution

Step 1: Test Infrastructure

python scripts/test_academic_collector.py
Validates database models, name analysis pipeline, and data quality. All tests should pass before proceeding.

Step 2: Pilot Collection

python -c "from collectors.academic_collector import AcademicCollector;
from app import app; app.app_context().push();
AcademicCollector().collect_pilot_sample()"
Collects bootstrap sample of 10-50 professors with full phonetic analysis. Tests collection pipeline.

Step 3: Run Pilot Analysis

python scripts/academic_deep_dive_analysis.py
Executes all 6 hypothesis tests, generates visualizations, creates comprehensive report. May be underpowered with small sample.

Step 4: Mass Collection (Production)

python scripts/collect_academics_mass_scale.py --phase 1
Scrapes top 50 universities for 10,000+ professors. WARNING: Slow due to rate limiting (3-6 weeks).

Step 5: Google Scholar Enrichment

python scripts/collect_academics_mass_scale.py --enrich-scholar
Batch enrichment with h-index and citations. Runs in 100-professor batches. 5 seconds per query = 8+ minutes per batch.

Step 6: Full Analysis

python scripts/academic_deep_dive_analysis.py
Re-run with full dataset. Generates publication-quality figures, comprehensive statistical results, and markdown report.
Timeline Warning
Full collection is slow by design due to polite rate limiting. Google Scholar allows ~5-10 queries/second. For 10,000 professors: 10,000 √ó 5 seconds = 50,000 seconds = 14 hours minimum. Spread collection over multiple days to avoid triggering anti-scraping measures. Pilot analysis (n=50) completes in under 1 hour.

Database Schema

Key fields: full_name, academic_rank, university_name, university_ranking, department, field_broad, field_specific
Indexes: university_name+rank, ranking, field, tier for fast queries
Relationships: One-to-one with AcademicAnalysis and AcademicResearchMetrics

Composites: intellectual_sophistication_score, academic_authority_composite

AcademicResearchMetrics Table

Purpose: Store Google Scholar research productivity data
Key fields: h_index, total_citations, i10_index, years_publishing, citations_per_year
Normalization: field_normalized_citations, field_normalized_h_index
Indexes: h_index, citations, productivity for H3 testing
